# ЭЛИДОМ — архитектурные решения

> Почему сделано так, а не иначе. Читать перед выбором подходов.

## Стек
- **Laravel 12** + **Filament 3** — бэкенд + админка. Стабильно, зрелое CMS-поведение.
- **Nuxt 3** (Vue 3 + SSR) — фронтенд. SSR обязателен по ТЗ (SEO-задел).
- **PostgreSQL 16** — БД. Выбрана за JSON-поля, полнотекстовый поиск, лёгкий путь в Supabase, если понадобится.
- **Nginx alpine** + **php-fpm 8.3** — стандартная пара под Laravel.
- **Docker Compose** (свой, не Sail) — Sail не дружит с Nuxt из коробки, нужен единый compose.

## Структура репозитория
```
eldom/
  backend/         Laravel + Filament
  frontend/        Nuxt
  prototype/       HTML-варианты (оставляем как референс)
  docker/          Dockerfile-ы и conf-ы
  docker-compose.yml
  .env             общие переменные для compose
```
Решение: backend/ + frontend/ внутри одного `eldom/`. Один compose поднимает всё. Подтверждено Денисом.

## Шрифты
- **Заголовки — Evolventa**, **текст — Ablation**.
- Внимание: исходно я перепутал местами; Денис поправил. Брендбук: head = Evolventa, body = Ablation.

## Палитра
Тёплый графит, не холодный чёрный. Оранжевый строго акцентный, **в одном месте** либо мелкими лейблами. Не размазывать.

## Формы
- На старте — отправка в **Telegram** (через bot API).
- Архитектура с интерфейсом-абстракцией, чтобы потом подменить на amoCRM/Ferico без переписывания.

## Что НЕ делаем
- Срок «на рынке» (1,5 года) в контенте.
- Сложные анимации, попапы.
- «Мы лучшие / №1» в текстах.
- Эконом/застройщики — не наша ЦА.

## Жёсткие правила фронта (от Дениса)
- **Только SVG-иконки.** Никаких эмодзи нигде — ни в UI, ни в текстах, ни в коде, ни в коммитах.
  Эмодзи запрещены, точка. Иконки — inline SVG (или sprite). Лежат в `frontend/app/components/icons/` или встроены в компоненты.
- **Mobile-first строго.** Базовая верстка пишется под `320–460px` и должна там быть идеальной — без сжатого текста, без вылезающих элементов, без лишних паддингов. Десктоп достраивается через `min-width` медиа.
- **Никаких лишних паддингов, отступов, обёрток.** Если блок можно построить без `<div class="wrapper">` — строим без. Каждый отступ должен быть оправдан.
- **Прелоадер обязателен.** Анимация на основе логотипа (`logo.svg` → клиентский SVG-mark, в виде чистых path). Перекрывает экран до готовности шрифтов и первой страницы.
