# ЭЛИДОМ — ТЗ на сайт

> Премиальный корпоративный сайт компании по натяжным потолкам.
> Сегмент — премиум. Гео — Москва и ближнее Подмосковье.
> Источники: бриф клиента (ответы владельца) + КП (Денис Романович).

---

## 1. О компании и позиционировании

| Параметр | Значение |
|---|---|
| Название | **ЭЛИДОМ** (на сайте пишем именно так) |
| Направление | Натяжные потолки + освещение |
| Гео | Москва и ближнее Подмосковье |
| Сегмент | **Премиум** |
| Цифры для сайта | **300+ объектов**, **15 000+ м²** |
| Что НЕ пишем | Срок на рынке (по факту 1,5 года — не звучит громко) |

**Суть премиума — не цена, а подход:**
- минимальное вовлечение заказчика в процесс (экономим его время);
- не озвучиваем проблемы — озвучиваем готовые варианты решения;
- не спрашиваем «какой свет нужен» — даём готовое решение;
- ценят за честность, человеческое отношение, прозрачные условия, чёткие сроки.

**Премиальный подход ощущается на каждом этапе:**
- предварительный расчёт = профессиональная прозрачная смета;
- выезд на замер = привозим мобильный шоурум (профили, полотна, освещение);
- финальная смета в любом удобном формате;
- на объекте — чистота и порядок, все поверхности укрываются для защиты чистовой отделки.

## 3. Структура сайта

### 3.1 Главная
- Премиальный первый экран по брендбуку (тёмная архитектурная эстетика со светом).
- Презентация компании.
- Ключевые цифры (300+ объектов, 15 000+ м²).
- Логичная последовательность блоков → вывод на заявку.

### 3.2 О компании
- Премиальный подход: мобильный шоурум, прозрачная смета, чистота на объекте, соблюдение сроков.
- То, за что ценят заказчики.

### 3.3 Услуги
- Виды потолков и что входит в работу.
- Без перегруза лишними деталями.

### 3.4 Портфолио
- **5–8 кейсов на старте**, пополняемые из админки.
- По каждому кейсу:
  - фото (профессиональная съёмка от клиента);
  - стоимость потолка;
  - стоимость освещения;
  - перечень работ **без углубления** (пример: «тканевые потолки JM на теневом профиле EuroSlott, освещение KraabSystems»);
  - **либо** развёрнутый кейс: какие задачи ставил заказчик и как реализовали.

### 3.5 Статьи (блог)
- Раздел блога.
- SEO-база заложена технически, чтобы развивать продвижение без переделки сайта.

### 3.6 Сотрудничество (партнёрка)
- Тезисно о партнёрке для дизайнеров и прорабов.
- **Условия (% и т.д.) НЕ раскрываем** — «хочешь узнать условия — заполни анкету».
- Анкета партнёра: **имя + телефон**.

## 5. Дизайн

### 5.1 Брендбук — цвета
| Цвет | HEX | Роль |
|---|---|---|
| Оранжевый | `#F97002` | **Акцент** (кнопки, ключевые элементы) — без перебора |
| Серый | `#A2A1A2` | Дополнительный (приглушённый текст) |
| Чёрный | `#000000` | Глубокий контраст |

**Рабочая палитра (брендбук + разбор Bork):**
| Токен | HEX | Применение |
|---|---|---|
| `--bg` тёплый графит | `#1F1B19` | основной фон (тёплый тёмный, НЕ холодный чёрный) |
| `--surface` | `#2B2724` | секции/панели |
| `--card` | `#39342F` | карточки |
| `--accent` | `#F97002` | оранжевый акцент |
| `--accent-grad` | `#C75A1E → #F97002` | тёплый янтарный градиент (точечно) |
| `--text` | `#F5F3F1` | тёплый белый |
| `--muted` | `#A2A1A2` | приглушённый текст |

> Оранжевый — **строго акцентный**: либо крупно в ОДНОМ месте (hero-градиент / CTA-плашка), либо мелкими лейблами. Не размазывать по экрану. Допустимы аккуратные тёплые градиенты.

### 5.2 Шрифты
| Назначение | Шрифт |
|---|---|
| Заголовки | **Ablation** |
| Основной текст | **Evolventa** |

### 5.3 Принципы
- Тёмная архитектурная эстетика «со светом».
- «Люкс люкс» — дорого с первого экрана.
- **Минимум анимации** (клиент подтвердил повторно). Только аккуратные микро-переходы, ничего, что тормозит/шумит.
- Без анимаций, которые тормозят сайт и создают визуальный шум.
- Никаких навязчивых попапов.
- Много «воздуха», галерейная подача (как в премиум-референсах).

### 5.4 Референсы клиента
| Сайт | Что берём |
|---|---|
| [bork.ru](https://www.bork.ru/) | Палитра и ощущение «дорого» (чёрный/графит, металлик, минимализм) |
| [shadoofdesign.ru](https://shadoofdesign.ru/) | Цветовые сочетания, последовательность блоков, подача кейсов |
| [denisromanovich.com](https://denisromanovich.com/) | Цветовые сочетания, чёткая последовательность блоков, отсутствие «воды» |
| (сайт с кейсами из брифа) | Оформление блока с кейсами |

**Разбор референсов (для дизайна):**
- **Shadoof:** тёмная/люкс эстетика, белый текст по тёмному, минимум акцентов. Порядок блоков: hero с видео → таглайн → услуги в 2 колонки → 4 ценности → «Почему мы» → портфолио карточками → процесс (6 шагов) → цитата основателя → форма → футер.
- **denisromanovich.com:** минималистичный тёмный/нейтральный, нумерация блоков (01–04), жирные заголовки-якоря, живые демо в портфолио, конверсионная структура (hero → услуги → проблема/кейс → цифры → тарифы → процесс → портфолио → квиз → FAQ → футер).
- **Bork (по скринам клиента):** база — **тёплый тёмный графит** (`~#2B2724`, не холодный чёрный), карточки чуть светлее (`~#39342F`). Оранжевый янтарный (`#C75A1E→#E8631E`) применён **смело, но в одном месте**: полноэкранный градиент в hero, ЛИБО одна CTA-плашка («Коллекция чемоданов»), ЛИБО мелкие лейблы «Новинка». Тонкая типографика, широкий letter-spacing в КАПСЕ, премиум-фото товара на тёмном, максимум «воздуха». Футер — приглушённые серые ссылки в 3–4 колонки.

## 6. Стек и архитектура

| Слой | Технология | Зачем |
|---|---|---|
| Фронтенд | **Nuxt** (Vue + SSR) | SEO-friendly, индексируется поисковиками |
| Бэкенд | **Laravel** | Управление контентом, обработка заявок, задел под CRM |
| Админка | **Filament** | Портфолио, блог, заявки из удобной панели |

**Что даёт стек:**
- чистый код и полный контроль над видом/поведением (критично для премиума);
- высокая скорость загрузки;
- техническая SEO-база сразу: SSR, semantic-разметка, мета, sitemap, schema;
- масштабируемость: легко дорастить до CRM и нового функционала.

> **Текущий приоритет (от Denis):** сначала базово **только фронт** — разные варианты дизайна. Админка (Filament) — позже.

## 7. SEO

- Упор на SEO сейчас **нет**, но техническая база обязательна.
- Цель: при желании развивать SEO **не делать новый сайт с нуля**.
- Закладываем: SSR, семантическую разметку, meta, sitemap.xml, schema.org.
